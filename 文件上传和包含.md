### 文件上传漏洞

服务器对上传的文件校验不严格，导致可以危害服务器的攻击性文件被上传。

### 文件包含漏洞

主要是使用“include”包含其他文件时，包含的文件可以被攻击者定义。

抓包观察有无明显的参数用于打开或使用文件，在漏洞挖掘过程中，注意下即可。如果遇到疑似文件包含，可以使用以下的测试方法。（因为不知道对参数有何处理，所以，该方法并不准确）

```bash
a.php是fileinclude.php的同一目录的文件
http://www.aaa.com/fileinclude.php?file=file://a.php
打开目标网站，一般不会被过滤掉
http://www.aaa.com/fileinclude.php?file=http://www.aaa.com
```

利用php伪协议读取敏感文件，详情请见[[008.win-敏感文件]]、[[lin-敏感文件]]、[[019.支持的协议和封装协议]]
常见的如：

```delphi
file://
http://
php://
```

被包含的文件无论是什么格式都会被当作时php文件执行，因为包含过来的是源代码，被包裹在了php标签中，所以被执行，如果包含的是php文件的话，会在原来的系统上先执行，然后将执行结果包含过来。

15.pass-21重构文件名数组，其中使用了白名单过滤。关键知识点，POST上传文件名时，可以上传文件名（即字符串），也可以上传一段【数组】。进行代码审计。

~~~ php
 $ext = end($file);
~~~

使用end（）获取到了上传的文件（通过explode（）函数，对.进行分割后形成的数组）最末尾的一部分。对取得的文件名后缀进行白名单过滤。

~~~ php
$file_name = reset($file) . '.' . $file[count($file) - 1];
~~~

然后使用上述代码对文件名重构，reset（）获取被分割后数组的第一部分。然后连接上数组长度-1为下标（长度为6的数组，即取下标为5的字符），也就是将这两部分拼接作为文件名。

漏洞分析：如果上传的文件名为一个数组，save_name[0]=gifma，save_name[2]=php，svae_name[3]=gif，其中没有下标为1的数据，

这样，end（）取到的文件名为gif，可以通过白名单的校验，reset（）取到的文件名为gifma（不重要），$file(count（$file）-1)会取到下标为2（长度为3，3-1=2）的文件名，由于上传的时候没有下标为1的值，所以原本能取到gif的，却取到了php，文件被重命名为了gif.php，还绕开了白名单，文件上传成功。

16.文件包含函数

1）include（）----包含文件，在用到文件时才加载，会在每一处出现调用的地方都重新调用(加载文件)，引入文件的时候，如果碰到错误，会给出提示，并继续运行下边的代码。

2）require（）----包含文件，文件一开始就加载，会在每一处出现调用的地方都重新调用(加载文件)，引入文件的时候，如果碰到错误，会给出提示，并停止运行下边的代码。

3）require_once（）------当含有 require_once 的php文件被执行时，则被 require_once 指定的外部文件都会包含进来，与其它的代码组成一个新的 php 文件，并开始执行。

4）include_once（）------当含有 include_once 的php文件被执行时，则被 include_once 指定的外部文件都会包含进来，与其它的代码组成一个新的 php 文件，并开始执行。

5）once（）指定的文件如果已经被包含过，则不会再次包含。它可以避免函数重定义，变量重新赋值等问题。

18.文件包漏洞中常用的伪协议
php://输入输出流
PHP提供了一些杂项输出流，允许访问PHP的输入输出流。标准输入输出和错误描述，内存中。磁盘备份的临时文件流以及可以操作其他读取写入文件资源的过滤器。
php://filter(本都磁盘文件进行读取)
用法：>filter=php://filter/cinvert.base64-encode/recource=xxx.php
条件：只是读取开始allow_url_fopen,不需要开始allow_url_include;

phar://伪协议
这个就是php解压缩报的一个函数，不管后缀是什么，都会当做压缩包来解压，用法：?file=phar://压缩包/内部文件 phar://xxx.png/shell.php 注意 PHP>=5.3.0压缩包需要是zip协议压缩，rar不行，将木马文件压缩后，改为其他任意格式的文件都可以正常使用。步骤：写一个一句话木马shell。php，然后用zip协议解压缩为shell.zip。然后将后缀改为png等其他格式
